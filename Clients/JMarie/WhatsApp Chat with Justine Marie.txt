5/15/26, 5:36 PM - Justine Marie: 1. What the Developer Must Do During the Build
The developer is responsible for building a clean, fast vehicle. Google rewards websites that provide an excellent user experience. During construction, he should be optimizing for:
Site Speed & Clean Code: Google uses page speed as a ranking factor. The developer needs to optimize image file sizes, minimize bulky code (CSS and JavaScript), and ensure fast server response times.
Mobile Responsiveness: The site must look and function flawlessly on phones. Google uses "mobile-first indexing," meaning it ranks your site based on how it performs on a mobile screen, not a desktop.
Clean URL Structure: The website links should be clean and readable (e.g., [mysite.com/services](https://mysite.com/services) instead of [mysite.com/page?id=123](https://mysite.com/page?id=123)). This needs to be set up as pages are created.
Site Architecture: Creating a logical flow of pages (a clear hierarchy) so search engine crawlers can easily map and understand your site.
2. What Is Done on the "Back End" (And Ongoing)
Once the structural foundation is laid, there is an interface—the Content Management System (CMS) backend—where SEO settings are managed. Both your developer during setup, and you (or an SEO specialist) later, will use the backend for On-Page SEO:
Meta Titles and Descriptions: The backend should have fields where you can type in the specific titles and snippets that show up in Google search results.
Alt Text for Images: Text descriptions attached to images so search engines know what the image portrays.
Sitemap Submission: Once the site goes live, a sitemap.xml file (a blueprint of your live site) needs to be generated from the backend and submitted to Google Search Console so Google knows you exist.
The Verdict: A Hybrid Approach
Your developer doesn't need to write your blog posts or do deep keyword research for your business strategy, but he must build an SEO-friendly foundation.
What to ask your developer:
"Are we implementing Technical SEO best practices during the build, like mobile optimization, clean site architecture, and fast page load speeds? And will I have backend access to easily update my own meta tags and alt text later?"
